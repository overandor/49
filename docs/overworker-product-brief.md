# Overworker Product Brief

## One-line definition

**Overworker is an AI work-execution platform that converts fragmented technical work into verified deliverables using persistent project memory, specialized worker modes, and deterministic quality gates.**

## Core problem

AI builders now generate more code, chats, repos, demos, prompts, and product ideas than they can package into credible assets. The bottleneck is not ideation. The bottleneck is completion, verification, and presentation.

Typical founder state:

```text
many repos
many chats
many demos
many files
many names
many unfinished product directions
few clean READMEs
few runnable demos
few investor-readable artifacts
few verified claims
```

## Core solution

Overworker ingests messy technical/project material and outputs a verified work package.

Inputs:

```text
GitHub repos
Hugging Face Spaces
chat archives
code fragments
product notes
technical claims
README drafts
demo links
screenshots
```

Outputs:

```text
product thesis
technical asset map
README/docs
risk disclosure
demo plan
investor one-pager
IP/moat summary
30/60/90 roadmap
Overwork Score
verification report
```

## Core workflow

```text
messy intent
→ project memory
→ task decomposition
→ worker routing
→ tool execution
→ verification firewall
→ deliverable export
```

### 1. Project Memory

Stores project-level context:

```text

```

### 2. Worker Router

Routes work to specialized workers:

```text
Research Worker

```

### 3. Verification Firewall

Before output is marked complete, Overworker checks:

```text
Does the file exist?
Does the code compile or run?
Are claims supported?
Are dependencies listed?
Is the output usable?
Is the risk disclosed?
Is there an explicit next action?
```

Core phrase:

> Work is not complete until it survives verification.
